# Chrome Web Store 등록 정보 — 한국어

## 항목 이름 (≤ 75자)

Earprint — YouTube Music 좋아요로 만드는 음악 별자리 · Taste DNA

## 간단한 설명 (≤ 132자)

YouTube Music 좋아요 곡을 음악 별자리 12원형, Taste DNA, 아티스트 지도, 개인 추천으로 풀어내는 취향 분석 도구.

## 자세한 설명 (≤ 16,000자)

**Earprint는 YouTube Music에서 본인이 직접 누른 "좋아요" 목록을 읽어, 알고리즘이 밀어주는 플레이리스트가 아니라 *내가 직접 골라온 음악*이 어떤 모습인지 보여줍니다.**

### 분석 결과로 받는 것

한 번 동기화 후 (1,500곡 기준 약 5~10분), Earprint 웹앱이 만들어내는 것:

- **음악 별자리 (Music Zodiac)** — 취향을 12개의 음악 원형 중 하나로 매칭 (예: Riff Architect, Velvet Collector, Dream Diver). 점성술이 아니라 라이브러리의 실제 신호(장르 분포·오디오 특성·앨범 몰입도·최근 변화)에서 도출.
- **Taste DNA** — 학술 근거 기반의 청취 패턴 분석: 익숙함 vs 새로움, 향수의 창(어느 연도가 취향의 중심인지), 장르 별자리, 오디오 특성 레이더.
- **인터랙티브 아티스트 지도** — 본인이 좋아한 아티스트들이 별자리처럼 펼쳐집니다. 드래그로 탐색, 아티스트 클릭하면 좋아하는 곡과의 공통점 표시, 추천을 끼워 넣어 빈 공간 채우기.
- **개인 곡 추천** — Tinder 스타일 평가 루프. 좋아요 한 곡은 라이브러리에 추가, 패스한 아티스트는 다음 추천에서 제외.
- **음악 월드컵** — 내 라이브러리에서 32/64/128/256강 토너먼트 (전체 라이브러리에서 랜덤 추출), 또는 장르 월드컵으로 "내가 인디팝을 더 좋아하나 시티팝을 더 좋아하나" 결판.

### 이 확장이 하는 일

확장은 의도적으로 최소한만 합니다:

1. 본인이 로그인된 브라우저 탭에서 YouTube Music "좋아요 한 음악" 페이지를 엽니다.
2. 확장 팝업의 "동기화" 버튼을 누릅니다.
3. 확장은 페이지를 스크롤하면서 YouTube가 이미 본인에게 보여주는 곡 — 곡명, 아티스트, 앨범, 좋아요 순서 — 을 읽습니다.
4. (웹사이트 첫 로그인 시 발급된) sync token으로 본인의 Earprint 계정에 그 행만 업로드합니다.

권한 모델은 이게 전부입니다.

### 이 확장이 절대 하지 않는 것

- **비밀번호 접근 없음.** Google·YouTube 비밀번호는 Earprint에 도달하지 않습니다. 로그인은 웹사이트의 표준 Google OAuth로 처리되고, 확장은 본인 계정에 묶인 토큰만 사용합니다.
- **시청·재생·검색 기록 안 봄.** 명시적으로 좋아요 누른 곡만 읽습니다.
- **백그라운드 스크래핑 없음.** 동기화는 사용자가 버튼을 누를 때만 발생합니다.
- **데이터 판매·광고 네트워크·추적 SDK 없음.** earprint.kwanho.dev/privacy 의 개인정보처리방침에 브라우저를 떠나는 모든 바이트가 명시되어 있습니다.
- **Append-only (추가 전용).** YouTube Music에서 좋아요를 취소해도 Earprint 기록에서 사라지지 않습니다 — Earprint는 "지금까지 좋아한 모든 것"의 영구 히스토리, 실시간 미러가 아닙니다.

### 왜 확장이 필요한가?

YouTube Music은 Spotify처럼 "내 좋아요 곡"을 읽을 수 있는 공개 API가 없습니다. 본인의 좋아요 목록을 읽는 유일한 방법은 music.youtube.com에 본인이 로그인한 상태에서 YouTube가 본인에게 이미 보여주는 페이지를 읽는 것입니다. 확장은 본인 탭에서 본인 클릭으로 그 일만 합니다.

### 가격

500곡까지는 누구나 무료. 유료 결제 시 더 큰 라이브러리, 전체 AI 음악 심리분석, 무제한 월드컵 등 잠금 해제. 한국은 원화, 그 외는 USD/EUR — **구독 없음**, 더 깊게 분석하고 싶을 때만 결제.

### 권한 요청 사유

- `storage` — sync token을 로컬에 저장해 매번 다시 붙여넣지 않게.
- `tabs` — 동기화 클릭 시 music.youtube.com 탭을 찾기 위해.
- `host_permissions: music.youtube.com` — 본인 탭의 "좋아요 한 음악" 페이지 읽기.
- `host_permissions: *.kwanho.dev` — 수집한 목록을 본인의 Earprint 계정에 업로드.

`<all_urls>`, `webRequest`, `cookies`, `history`, `bookmarks` 등 위 두 호스트 외 권한은 일절 요청하지 않습니다.
